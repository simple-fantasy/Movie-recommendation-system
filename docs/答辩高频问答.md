# 答辩高频问答（建议口径）

## 1. 为什么选择 ItemCF 作为主模型？
ItemCF 的工程稳定性和可解释性更好，适合本项目的数据规模与毕设时间窗口。它可以离线预计算相似度，在线只做查表与加权排序，响应快且容易解释推荐来源。

## 2. 为什么还做 NCF 和 Hybrid？
这是扩展探索：希望验证“传统协同过滤 + 神经网络重排”的可行性。最终实验中 ItemCF 仍是稳健基线，NCF/Hybrid 作为优化尝试和对照实验。

## 3. 你的创新点是什么？
本项目创新点属于系统集成与工程实现层面：
- 完整的推荐闭环（数据导入、训练、在线推荐、评估、可视化）
- 推荐解释机制（基于相似电影贡献）
- 推荐反馈机制（like/dislike）
- 多模型对比与消融实验流程

## 4. 如何解决冷启动？
对于无历史评分用户，采用热门高分电影兜底；用户产生评分后再切换个性化推荐。后续可引入内容特征做更细粒度冷启动。

## 5. 为什么不用在线 A/B 测试？
本项目是本科毕设，场景与流量条件有限，优先完成离线指标闭环。在线 A/B 是后续工程化方向。

## 6. 如何保证评估公平？
采用统一的 leave-last-out 思路进行离线评估；并在多模型脚本中固定参数进行对比与消融。结论不夸大，明确当前最优基线。

## 7. 如果老师问“为什么 NCF 没有更好”？
可以回答：数据规模、特征维度和训练协议都会影响深度模型效果。在当前数据条件下，ItemCF 的统计归纳更稳健；NCF 价值主要体现在扩展性与后续潜力。

## 8. 工业化差距在哪里？
- 缺少实时特征与在线学习
- 缺少多路召回和复杂排序链路
- 缺少在线实验平台与监控闭环

## 9. 下一步优化方向？
- 引入更严格时间切分与评估协议
- 增强冷启动（内容特征、混合召回）
- 增加在线反馈学习机制
- 完善自动化测试与部署规范
